Deployment Ready - Follow These 3 Steps

✅ Good News! Your OmniMerge application is completely built and ready to deploy to GitHub. Just follow the steps below!
1️⃣

Get Your GitHub Personal Access Token (2 minutes)

You need a token to allow the deployment script to create and push to GitHub:

1. Go to: https://github.com/settings/tokens 2. Click: "Generate new token" → "Generate new token (classic)" 3. Token name: OmniMerge-Deployment 4. Select scopes: ✅ repo (Full control) 5. Click: "Generate token" at bottom 6. COPY the token (you won't see it again!)

Keep this token safe! You'll use it in Step 2.

2️⃣

Run the Deployment Script (1 minute)

Choose your OS:

🪟 Windows (PowerShell):

cd "C:\Users\Finance-MGR\Downloads\Stock-Data-Vault\Stock-Data-Vault - Phase 2\artifacts\consolidation-tool" .\deploy-to-github.ps1

🍎 Mac / 🐧 Linux (Bash):

cd artifacts/consolidation-tool chmod +x deploy.sh ./deploy.sh
3️⃣

Enter Your Information (Automatic after this!)

When the script runs, it will ask for:

  • GitHub Username: Your GitHub username (e.g., notrahul98)
  • GitHub Token: The token you created in Step 1

The script will then automatically:

  • ✅ Create a repository named "OmniMerge" on your GitHub
  • ✅ Push all code and files
  • ✅ Enable GitHub Pages
  • ✅ Make your app live!

❓ Troubleshooting

Problem Solution
Script won't run Check PowerShell execution policy: Set-ExecutionPolicy -ExecutionPolicy RemoteSigned
Invalid token error Token expired or incorrect. Create a new one at github.com/settings/tokens
Repository already exists Use a different repo name or delete existing one from GitHub Settings
Pages not loading Wait 1-2 minutes, clear cache (Ctrl+Shift+Delete), then refresh
